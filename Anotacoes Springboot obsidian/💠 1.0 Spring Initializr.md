## 💠 O que é Spring Initializr?

É um site disponibilizado pela própria equipe que criou o Spring.

Esse site é útil pelo fato de que com ele podemos criar nosso projeto com as dependências necessárias, sem ter que passar por burocracia chata.

Ele pode ser acessado em https://start.spring.io/; e tem essa UI:

![[{BFE03377-F8EF-42BE-8291-6C834DCAFC42} 1.png]]

 💠 O primeiro passo é escolher uma opção de gerenciador de dependências. Nesse aqui, utilizaremos o [[💠 2.0 Maven]].

💠 Logo após isso, escolhemos a linguagem. Nesse projeto utilizaremos **Java**.

💠 Depois de selecionar a linguagem, devemos escolher a versão do **Springboot** que utilizaremos. É recomendado utilizar a versão mais recente que não seja SNAPSHOT, pois é mais estável.

💠 No campo **Group** você deve colocar o nome da empresa.

💠 No campo **Artifact** você deve colocar o nome do projeto.

💠 **Packaging** você deve selecionar **Jar**, **War** é coisa antiga, para servidores **Tomcat**.

💠 Configuration você deve selecionar Properties. É mais simples.

💠 Versão do Java deve ser a **21**. Porque a Oracle lança uma versão nova do Java a cada 6 meses, mas a maioria delas recebe suporte por apenas 6 meses e depois é abandonada. As versões LTS são especiais — recebem atualizações de segurança e correções por anos.

Java 17 — LTS (suporte até 2029)
**Java 21 — LTS (suporte até 2031) ✅**
Java 25 — LTS (vai sair em setembro/2025)
Java 26 — versão comum, sem suporte estendido

O 25 ainda não saiu quando o Spring Initializr foi atualizado, e o 26 não tem suporte longo. Para um projeto sério, sempre prefira uma versão LTS estável.
Java 21  é o LTS mais recente e consolidado, com amplo suporte em bibliotecas, tutoriais e no mercado de trabalho.
