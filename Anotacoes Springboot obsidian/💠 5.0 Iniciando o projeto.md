## 💠 Implementando um link para o nosso banco de dados

Primeiramente, vamos acessar um arquivo de configuração do nosso projeto, para fazê-lo reconhecer nosso banco de dados.

Esse arquivo é o `application.properties`, que se localiza em:

	`pasta_do_projeto > src > main > resources > application.properties`


Devemos inserir as seguintes linhas nesse arquivo:

``` 
spring.datasource.url = jdbc:postgresql://localhost:5432/product-api
spring.datasource.username = postgres
spring.datasource.password = 1234
```

Com isso, estamos inserindo link e informações a respeito do banco de dados que estaremos utilizando no nosso projeto.

## 💠 Criando o banco de dados no pgAdmin

O pgAdmin4 é uma interface para criar bancos de dados PostgreSQL.

Para iniciar nosso banco de dados, devemos ir em:

`Servers > PostgreSQL > Databases > Create > Database`

![[{3443B89A-AF91-4334-AFCF-4FADEE310215}.png]]

Depois disso, basta dar um nome à base de dados, igual ao que foi definido no arquivo `application.properties`, e clicar em salvar. A base de dados será criada instantaneamente.

## 💠 Criando as entidades modelos do BD no Spring

Ao invés de criar os produtos do nosso CRUD (Que serão produtos de supermercado) no pgAdmin, faremos isso diretamente com o Spring, dentro do projeto.

Para isso, iremos em:

	 `pasta_do_projeto > src > main > java > com.cosmodev.crud1`

E dentro dessa pasta, criaremos um novo pacote chamado `model`.

![[{916C4127-797F-4C7B-AC7E-46FED97F3732}.png]]

Dentro desse pacote, criaremos as classes de entidades modelo do nosso BD. Começaremos com a classe `product`.

![[{43BDD82A-4B58-4DC9-828C-E86D49549125}.png]]

Dentro da nossa classe `Product`, criamos os atributos, getters, setters e construtor:

```java
package com.cosmodev.crud1.model;  
  
public class Product {  
  
    //  Product class atributes  
    //  Atributos da classe produto    private Integer id;  
    private String name;  
    private Long price;  
  
    // Constructor  
  
    public Product(Integer id, String name, Long price) {  
        this.id = id;  
        this.name = name;  
        this.price = price;  
    }  
  
    // Getters and Setters  
  
    public Integer getId() {  
        return id;  
    }  
  
    public void setId(Integer id) {  
        this.id = id;  
    }  
  
    public String getName() {  
        return name;  
    }  
  
    public void setName(String name) {  
        this.name = name;  
    }  
  
    public Long getPrice() {  
        return price;  
    }  
  
    public void setPrice(Long price) {  
        this.price = price;  
    }  
}
```

Após a classe ficar pronta, usaremos [[💠5.1 Anotações no Spring]] para definir a natureza de nossa classe no contexto do projeto.

Na linha anterior à declaração da classe, adicionaremos as seguintes linhas:

```java
@Entity(name = "product")
@Table(name = "product")
public class Product {
```

- **@Entity** - Informa ao Spring que a classe a seguir se trata de uma entidade modelo para o BD, fazendo ele reconhecê-la como tal. No parâmetro, definiremos o nome.

- **@Table** - Informa ao Spring que deve ser criada uma tabela no BD para o elemento alvo. No parâmetro, definiremos o nome.

Também adicionaremos anotações a um atributo de nossa classe, o ID:

```java
@Id  
@GeneratedValue(strategy = GenerationType.AUTO)  
private Integer id;  
private String name;  
private Long price;
```

- **@Id** - Informa ao Spring que o atributo a seguir é um Id de um objeto no BD.

- **@GeneratedValue(strategy = GenerationType.AUTO)** - Informa ao Spring que esse valor será criado de forma automática, por se tratar de um Id.
