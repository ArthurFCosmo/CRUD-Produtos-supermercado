Seguindo o fluxo, vamos criar o repository do nosso projeto.

No IntelliJ, vamos na pasta do projeto e criaremos um novo pacote.

> `right click na pasta > new > package > com.cosmodev.crud1.repositores`

O nome deve ser **repositories**, no plural mesmo.

---

Dentro desse pacote, devemos criar uma interface com o nome da nossa entidade (ProductRepository):

> `right click no pacote > new > Java class > Interface > ProductRepository`

---

Na declaração dessa interface, devemos torná-la filha de `JpaRepository<>`.

```JAVA
import org.springframework.data.jpa.repository.JpaRepository;  
  
public interface ProductRepository extends JpaRepository<T , ID> {  
}
```

⚠️ **ATENÇÃO ÀS IMPORTAÇÕES**

Para cada entidade que mapearmos no nosso projeto, deveremos criar um `Repository` desse para ela para que ela possa entrar em contato com o BD. Fazemos isso por que quando criamos essa interface filha de `JpaRepository<>`, essa superclasse já possui os métodos integrados  para fazer as interações com o BD.

Nos parâmetros dessa superclasse, devemos declarar primeiro (T) qual a classe dessa entidade que vamos instanciar (Product), e depois o tipo do ID (ID), que é (Integer).

```java
public interface ProductRepository extends JpaRepository<Product, Integer> {  
}
```
